# Learn Skill — 快速学科学习技能设计规范

## 概述

为大学生打造一个 Claude Code 技能（`/learn`），帮助学生快速学习理工科科目。学生自主选择深度学习或考试冲刺模式，支持上传课件/教材/链接作为学习材料，多轮对话教学为主、一键生成材料为辅，搭配本地 HTML 可视化页面展示公式、思维导图、闪卡和进度。

## 设计目标

1. **双模式**：深度学习（理解本质+项目实战）和考试冲刺（考点提取+题海战术），学生自主切换
2. **全材料支持**：接受纯文本主题名、PDF 课件、截图、课程网站链接、考试大纲
3. **混合交互**：默认多轮互动教学，可随时要求生成完整材料
4. **可视化**：本地 HTML 页面渲染 KaTeX 公式、Mermaid 思维导图、闪卡、进度看板、错题本
5. **持久化**：学习进度可断点续学，跨会话恢复
6. **理工科为主**：侧重数学、物理、计算机等学科

## 模块详细设计

### 1. 入口 Skill（`skill.md`）

**职责**：信息收集 → 材料解析 → 模式分发。不做教学。

#### 阶段 1：信息收集

学生输入 `/learn` 后，逐一确认以下信息：

1. **学科确认** — 确认学科名称
2. **模式确认** — 深度学习 vs 考试冲刺
3. **材料收集** — 接收粘贴的 URL、拖入的 PDF/截图文件
4. **范围确认** — 整门课还是特定章节
5. **时间约束**（可选）— 影响内容密度和教学节奏

#### 阶段 2：材料解析

对所有材料做结构化处理，输出「学情摘要」写入 `progress.json`：

```json
{
  "subject": "信号与系统",
  "mode": "sprint",
  "materials": [
    {"type": "pdf", "path": "课件.pdf", "summary": "覆盖第1-8章"},
    {"type": "url", "url": "https://...", "summary": "课程大纲..."}
  ],
  "constraints": "3天内，每天约4小时",
  "exam_scope": "第1-6章，闭卷，题型：选择+计算+证明"
}
```

#### 阶段 3：模式分发

依据学生选的模式，调用对应子 skill 并传入学情摘要。

---

### 2. 深度学习子 Skill（`deep.md`）

**目标**：理解本质、建立直觉、实战验证。

#### 教学流程（6 步）

```
第0步：摸底测试 → 测试当前水平，发现数学基础缺口
第1步：建立动机 → "这个概念解决了什么问题？"
第2步：逐步推导 → 从第一性原理推导，直觉 → 数学 → 直觉
第3步：苏格拉底反问 → "如果去掉这个条件会怎样？"
第4步：概念检验 → 1-2 道理解题，验证是否真懂
第5步：项目实战 → 生成可执行的代码项目框架
第6步：跨域关联 → 这个概念在其他领域怎么用的？
```

#### 铁律

| 原则 | 说明 |
|------|------|
| 不准直接给答案 | 先反问学生思路，再引导 |
| 直觉优先 | 公式前先用日常语言建立直觉 |
| 不透不推进 | 每节末尾确认理解后才进下一节 |
| 项目要真做 | 生成可执行的项目框架而非"请自行练习" |
| 缺口主动补 | 发现前置知识不足时主动回溯 |

#### 输出产物

- `notes.md`：学生自己语言写的理解、推导关键步骤
- `progress.json`：掌握度评估（1-5）、已完成章节
- 项目代码文件（如 `.py` / `.m`）

---

### 3. 考试冲刺子 Skill（`sprint.md`）

**目标**：精准定位考点、快速记忆、题海碾压、错题闭环。

#### 教学流程（5 步）

```
第1步：划范围 → 提取考点清单，标注频次/难度/题型
第2步：速通   → 精简讲解 + 记忆口诀 + 生成闪卡 → cards.json
第3步：刷题   → 按考点出题，从易到难三档
第4步：复盘   → 错题收集，分析错因分类
第5步：补刀   → 薄弱考点生成同类题，直到正确率达标
```

#### 考点数据格式

```json
{
  "exam_points": [{
    "id": "ss-03",
    "topic": "傅里叶变换的性质",
    "weight": "高频",
    "difficulty": "中等",
    "question_types": ["选择", "计算"],
    "flashcard": {
      "front": "时域卷积对应频域什么？",
      "back": "乘积。$f(t) * g(t) \\leftrightarrow F(\\omega)G(\\omega)$"
    },
    "mastery": 0
  }]
}
```

#### 铁律

| 原则 | 说明 |
|------|------|
| 不拓展 | 考试不考的绝对不讲 |
| 先骨架后血肉 | 先给考点全局地图，再逐个击破 |
| 题量堆熟练度 | 用题海而非难题加深理解 |
| 错题不过夜 | 每道错题当场复盘 + 同类题重做 |

#### 与深度模式的区别

| 维度 | 深度学习 | 考试冲刺 |
|------|---------|---------|
| 目标 | 理解本质，建立体系 | 拿分，快速应试 |
| 节奏 | 慢，学生主导 | 快，考点驱动 |
| 提问风格 | "你觉得为什么？" | "这个概念记住的口诀是？" |
| 输出 | 笔记 + 项目代码 | 闪卡 + 模拟卷 + 错题本 |

---

### 4. 共享规范（`shared.md`）

定义跨模块约定，供所有子 skill 引用：

- **输出格式规范**：`progress.json`、`cards.json`、`notes.md` 的 schema
- **可视化页面协议**：数据文件写入路径和格式约定
- **文件读写规范**：统一使用绝对路径，JSON 用标准缩进
- **学科名称规范**：目录名使用中文全称，不含特殊字符

---

### 5. 可视化页面（`index.html`）

本地单文件 HTML，使用 CDN 引入依赖。skill 负责生成数据文件，页面负责渲染展示。

#### 四大 Tab

| Tab | 功能 | 技术 |
|-----|------|------|
| 考点地图 | 思维导图，节点按掌握度着色 | Mermaid.js |
| 闪卡翻页 | 正反面翻转动画，左右翻页 | 原生 JS + CSS 3D |
| 进度看板 | 考点覆盖率、正确率、用时统计 | Chart.js |
| 错题本 | 错题列表 + 错因标签 + 展开解析 | 原生 JS |

#### 技术选型

| 能力 | 方案 |
|------|------|
| 公式渲染 | KaTeX（CDN） |
| 思维导图 | Mermaid.js（CDN，声明式） |
| 闪卡交互 | 原生 JS + CSS 3D flip |
| 进度图表 | Chart.js（CDN） |
| 数据来源 | 同目录 `cards.json` / `progress.json`，通过 `fetch` 加载 |

#### 与 Skill 的协作

- skill 每次教学结束后，将 `progress.json` 和 `cards.json` **同时写入** `learn-sessions/{科目}/`（数据源）和 `visualize/data/`（可视化镜像）
- 学生用浏览器打开 `index.html`，页面通过 `fetch('./data/progress.json')` 加载数据，同目录读取无跨域问题
- 两者通过文件系统解耦，skill 只管写，页面只管读

### 6. 会话与持久化

#### `progress.json` schema

```json
{
  "subject": "信号与系统",
  "mode": "sprint",
  "created_at": "2026-06-05",
  "updated_at": "2026-06-05T14:30:00",
  "status": "in_progress",
  "current_stage": "刷题_第3章",
  "exam_points": [
    {
      "id": "ss-01",
      "topic": "信号分类",
      "mastery": 5,
      "total_questions": 8,
      "correct": 8
    }
  ],
  "weak_points": ["ss-03"],
  "material_summary": "已解析：课件.pdf（第1-6章）"
}
```

#### 会话恢复流程

1. 学生输入 `/learn 信号与系统`
2. 入口 skill 读取 `learn-sessions/信号与系统/progress.json`
3. 发现 `status: "in_progress"` → 提示断点位置与掌握度概况
4. 学生选择"继续" → 子 skill 从断点恢复，错题优先复习

#### 跨学科索引

`learn-sessions/_index.json`：
```json
[
  {
    "subject": "信号与系统",
    "mode": "sprint",
    "status": "in_progress",
    "updated": "2026-06-05T14:30:00"
  }
]
```
支持 `/learn --list` 查看所有学习进度。

#### 数据量控制

- 单个 `cards.json` 上限约 200 张闪卡，超出提示归档
- `notes.md` 不设限
- 文件总大小控制在 1MB 以内确保读写效率

---

## 非功能需求

- skill 的 token 消耗需合理控制，单轮对话长度不超过 2000 token（教学响应部分）
- HTML 页面首次加载 < 3 秒（CDN 资源）
- 支持 Windows/macOS/Linux 跨平台使用
- 可视化页面需在 Chrome/Firefox/Edge 上正常工作
